Pertimbangan-pertimbangan

Saat Anda bekerja dengan fitur kueri lintas-database di Amazon Redshift, pertimbangkan hal berikut:

  • Amazon Redshift mendukung kueri lintas-database pada tipe node ra3.4xlarge, ra3.16xlarge, dan ra3.xlplus.

  • Amazon Redshift mendukung penggabungan data dari tabel atau tampilan di satu atau beberapa database dalam klaster Amazon Redshift yang sama.

  • Amazon Redshift Tanpa Server mendukung kemampuan lintas database yang sama dengan klaster Amazon Redshift, sehingga Anda dapat menggabungkan data dari tabel atau tampilan di satu atau beberapa database dalam ruang nama tanpa server.

  • Semua query dalam transaksi pada database yang terhubung membaca data dalam keadaan yang sama dari database lain sebagai data pada awal transaksi. Pendekatan ini membantu untuk memberikan konsistensi transaksional query di seluruh database. Amazon Redshift mendukung konsistensi transaksional untuk kueri lintas-database.

  • Untuk mendapatkan metadata di seluruh database, gunakan tampilan metadata SVV_ALL* dan SVV_REDSHIFT*. Anda tidak dapat menggunakan notasi tiga bagian atau skema eksternal untuk menanyakan tabel metadata lintas database atau tampilan di bawah information_schema dan pg_catalog.

Keterbatasan:

Saat Anda bekerja dengan fitur kueri lintas-database di Amazon Redshift, perhatikan batasan berikut:

  • Ketika Anda query objek database pada database yang tidak terhubung lainnya, Anda memiliki akses membaca hanya untuk objek-objek database.

  • Anda tidak dapat meminta tampilan yang dibuat pada database lain yang merujuk ke objek dari database lain.

  • Anda hanya dapat membuat tampilan akhir yang mengikat dan terwujud pada objek database lain di klaster. Anda tidak dapat membuat tampilan reguler pada objek database lain di klaster.

  • Amazon Redshift tidak mendukung tabel dengan hak istimewa tingkat kolom untuk kueri lintas-database.

  • Amazon Redshift tidak mendukung penskalaan konkurensi untuk kueri yang membaca data dari database lain.

  • Amazon Redshift tidak mendukung objek katalog kueri padaAWS Glue atau database federasi. Untuk query benda-benda ini, pertama membuat skema eksternal yang mengacu pada sumber-sumber data eksternal di setiap database.

  • Amazon Redshift tidak mendukung cache hasil untuk kueri basis data.
