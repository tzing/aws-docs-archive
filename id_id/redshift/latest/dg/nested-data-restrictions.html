Pembatasan data

Pembatasan berikut ini berlaku untuk data yang di-

  • Array hanya dapat berisi skalar ataustruct jenis. Arrayjenis tidak dapat berisiarray ataumap jenis.

  • Redshift Spectrum mendukung tipe data yang kompleks hanya sebagai tabel eksternal.

  • Query dan subquery hasil kolom harus skalar.

  • JikaOUTER JOIN ekspresi mengacu pada tabel bersarang, dapat merujuk hanya ke tabel itu dan array bersarang (dan peta). JikaOUTER JOIN ekspresi tidak merujuk ke tabel bersarang, dapat merujuk ke sejumlah tabel non-bersarang.

  • JikaFROM klausa dalam subquery mengacu pada tabel bersarang, itu tidak dapat merujuk ke tabel lainnya.

  • Jika subquery tergantung pada tabel bersarang yang mengacu pada orang tua, Anda dapat menggunakan induk hanya dalamFROM klausa. Anda tidak dapat menggunakan induk dalam klausa lain, sepertiWHERE klausaSELECT or. Misalnya, kueri ini tidak dijalankan. 

    
SELECT c.name.given 
FROM   spectrum.customers c 
WHERE (SELECT COUNT(c.id) FROM c.phones p WHERE p LIKE '858%') > 1

    Query berikut bekerja karena induk hanyac digunakan dalamFROM klausa subquery.

    
SELECT c.name.given 
FROM   spectrum.customers c 
WHERE (SELECT COUNT(*) FROM c.phones p WHERE p LIKE '858%') > 1

  • Sebuah subquery yang mengakses data bersarang di mana saja selainFROM klausa harus mengembalikan nilai tunggal. Satu-satunya pengecualian adalah(NOT) EXISTS operator dalamWHERE klausa.

  • (NOT) IN tidak didukung.

  • Kedalaman bersarang maksimum untuk semua jenis bersarang adalah 100. Pembatasan ini berlaku untuk semua format file (Parket, ORC, Ion, dan JSON).

  • Subqueries agregasi yang mengakses data bersarang hanya dapat merujuk kearrays danmaps dalamFROM klausul mereka, bukan ke tabel eksternal.

  • Menanyakan pseudocolumns data bersarang dalam tabel Redshift Spectrum tidak didukung. Untuk informasi selengkapnya, lihat Pseudocolumns.
