Membuat skema eksternal untuk Amazon Redshift Spectrum

Semua tabel eksternal harus dibuat dalam skema eksternal, yang Anda buat menggunakanBUAT SKEMA EKSTERNALpernyataan.

catatan

Beberapa aplikasi menggunakan istilahbasis datadanskemasecara bergantian. Di Amazon Redshift, kami menggunakan istilahskema.

Skema eksternal Amazon Redshift mereferensikan database eksternal dalam katalog data eksternal. Anda dapat membuat database eksternal di Amazon Redshift, diAthena, diAWS Glue Data Catalog, atau dalam metastore Apache Hive, sepertiAmazon ESDM. Jika Anda membuat database eksternal di Amazon Redshift, database berada di Katalog Data Athena. Untuk membuat database dalam metastore Hive, Anda perlu membuat database dalam aplikasi Hive Anda.

Amazon Redshift memerlukan otorisasi untuk mengakses Katalog Data di Athena dan file data di Amazon S3 atas nama Anda. Untuk memberikan otorisasi itu, Anda terlebih dahulu membuatAWS Identity and Access Management(IAM) peran. Kemudian Anda melampirkan peran ke klaster Anda dan memberikan Amazon Resource Name (ARN) untuk peran di Amazon RedshiftCREATE EXTERNAL SCHEMApernyataan. Untuk informasi selengkapnya tentang otorisasi, lihatKebijakan IAM untuk Amazon Redshift Spectrum.

catatan

Jika saat ini Anda memiliki tabel eksternal Redshift Spectrum di Katalog Data Athena, Anda dapat memigrasi Katalog Data Athena Anda keAWS GlueKatalog Data. Untuk menggunakanAWS GlueKatalog Data dengan Redshift Spectrum, Anda mungkin perlu mengubah kebijakan IAM Anda. Untuk informasi lebih lanjut, lihatUpgrade keAWS GlueKatalog Datadi dalamPanduan Pengguna Amazon Athena.

Untuk membuat database eksternal pada saat yang sama Anda membuat skema eksternal, tentukanFROM DATA CATALOGdan termasukCREATE EXTERNAL DATABASEklausul diCREATE EXTERNAL SCHEMApernyataan.

Contoh berikut membuat skema eksternal bernamaspectrum_schemamenggunakan database eksternalspectrum_db.

create external schema spectrum_schema from data catalog 
database 'spectrum_db' 
iam_role 'arn:aws:iam::123456789012:role/MySpectrumRole'
create external database if not exists;

Jika Anda mengelola katalog data Anda menggunakan Athena, tentukan nama database Athena danAWSWilayah di mana Katalog Data Athena berada.

Contoh berikut membuat skema eksternal menggunakan defaultsampledbdatabase di Katalog Data Athena.

create external schema athena_schema from data catalog 
database 'sampledb' 
iam_role 'arn:aws:iam::123456789012:role/MySpectrumRole' 
region 'us-east-2';
catatan

Yangregionparameter referensiAWSWilayah di mana Katalog Data Athena berada, bukan lokasi file data di Amazon S3.

Jika Anda mengelola katalog data menggunakan metastore Hive, seperti Amazon EMR, grup keamanan Anda harus dikonfigurasi untuk mengizinkan lalu lintas antar klaster.

Dalam pernyataan CREATE EXTERNAL SCHEMA, tentukanFROM HIVE METASTOREdan termasuk URI metastore dan nomor port. Contoh berikut membuat skema eksternal menggunakan database metastore Hive bernamahive_db.

create external schema hive_schema
from hive metastore
database 'hive_db'
uri '172.10.10.10' port 99
iam_role 'arn:aws:iam::123456789012:role/MySpectrumRole'

Untuk melihat skema eksternal untuk klaster Anda, kueri tabel katalog PG_EXTERNAL_SCHEMA atau tampilan SVV_EXTERNAL_SCHEMAS. Contoh berikut query SVV_EXTERNAL_SCHEMAS, yang bergabung PG_EXTERNAL_SCHEMA dan PG_NAMESPACE.

select * from svv_external_schemas

Untuk sintaks dan contoh perintah lengkap, lihatBUAT SKEMA EKSTERNAL.

Bekerja dengan katalog eksternal di Amazon Redshift Spectrum

Metadata untuk database eksternal Amazon Redshift Spectrum dan tabel eksternal disimpan dalam katalog data eksternal. Secara default, metadata Redshift Spectrum disimpan dalam Katalog Data Athena. Anda dapat melihat dan mengelola database dan tabel Redshift Spectrum di konsol Athena Anda.

Anda juga dapat membuat dan mengelola database eksternal dan tabel eksternal menggunakan bahasa definisi data Hive (DDL) menggunakan Athena atau metastore Hive, seperti Amazon EMR.

catatan

Kami merekomendasikan penggunaan Amazon Redshift untuk membuat dan mengelola database eksternal dan tabel eksternal di Redshift Spectrum.

Melihat database Redshift Spectrum di Athena danAWS Glue

Anda dapat membuat database eksternal dengan menyertakan CREATE EXTERNAL DATABASE IF NOT EXISTS klausa sebagai bagian dari pernyataan CREATE EXTERNAL SCHEMA Anda. Dalam kasus tersebut, metadata database eksternal disimpan dalam Katalog Data Anda. Metadata untuk tabel eksternal yang Anda buat memenuhi syarat oleh skema eksternal juga disimpan dalam Katalog Data Anda.

Athena danAWS Gluememelihara Katalog Data untuk setiap didukungWilayah AWS. Untuk melihat metadata tabel, log on ke Athena atauAWS Gluekonsol. Di Athena, pilihSumber data, AndaAWS Glue, kemudian melihat rincian database Anda. DalamAWS Glue, pilihDatabase, database eksternal Anda, kemudian melihat rincian database Anda.

Jika Anda membuat dan mengelola tabel eksternal Anda menggunakan Athena, daftarkan database menggunakan CREATE EXTERNAL SCHEMA. Misalnya, perintah berikut mendaftarkan database Athena bernamasampledb.

create external schema athena_sample
from data catalog
database 'sampledb'
iam_role 'arn:aws:iam::123456789012:role/mySpectrumRole'
region 'us-east-1';

Ketika Anda query tampilan sistem SVV_EXTERNAL_TABLES, Anda melihat tabel di Athenasampledbdatabase dan juga tabel yang Anda buat di Amazon Redshift.

select * from svv_external_tables;

Mendaftarkan database metastore Apache Hive

Jika Anda membuat tabel eksternal dalam metastore Apache Hive, Anda dapat menggunakan CREATE EXTERNAL SCHEMA untuk mendaftarkan tabel tersebut di Redshift Spectrum.

Dalam pernyataan CREATE EXTERNAL SCHEMA, tentukan FROM HIVE METASTORE klausa dan memberikan Hive metastore URI dan nomor port. Peran IAM harus menyertakan izin untuk mengakses Amazon S3 tetapi tidak memerlukan izin Athena apa pun. Contoh berikut register metastore Hive. 

create external schema if not exists hive_schema
from hive metastore
database 'hive_database'
uri 'ip-10-0-111-111.us-west-2.compute.internal' port 9083 
iam_role 'arn:aws:iam::123456789012:role/mySpectrumRole';

Mengaktifkan klaster Amazon Redshift Anda untuk mengakses klaster Amazon EMR Anda

Jika metastore Hive Anda ada di Amazon EMR, Anda harus memberikan akses klaster Amazon Redshift ke klaster Amazon EMR Anda. Untuk melakukannya, Anda membuat grup keamanan Amazon EC2. Anda kemudian mengizinkan semua lalu lintas masuk ke grup keamanan EC2 dari grup keamanan klaster Amazon Redshift Anda dan grup keamanan klaster Amazon EMR Anda. Kemudian Anda menambahkan keamanan EC2 ke klaster Amazon Redshift dan klaster Amazon EMR Anda.

Melihat nama grup keamanan klaster Amazon Redshift

Untuk menampilkan grup keamanan, lakukan hal berikut:

  1. Masuklah ke AWS Management Console dan buka konsol Amazon Redshift di https://console.aws.amazon.com/redshift/.

  2. Pada menu navigasi, pilihCluster, lalu pilih cluster dari daftar untuk membuka detailnya.

  3. PilihPropertidan melihatPengaturan jaringan dan keamananbagian.

  4. Temukan grup keamanan Anda diGrup keamanan VPCdan perhatikan itu.

Melihat nama grup keamanan node master Amazon EMR

  1. Buka klaster Amazon EMR Anda. Untuk informasi lebih lanjut, lihatMenggunakan konfigurasi keamanan untuk mengatur keamanan klasterdi dalamPanduan Manajemen Amazon EMR.

  2. Di bawahKeamanan dan akses, catat nama grup keamanan node master Amazon EMR.

Untuk membuat atau memodifikasi grup keamanan Amazon EC2 untuk memungkinkan koneksi antara Amazon Redshift dan Amazon EMR

  1. Di dasbor Amazon EC2, pilihGrup keamanan. Untuk informasi lebih lanjut, lihatAturan grup keamanandi dalamPanduan Pengguna Amazon EC2 untuk Instans Linux

  2. Pilih Create security group (Buat grup keamanan).

  3. Jika Anda menggunakan VPC, pilih VPC tempat klaster Amazon Redshift dan Amazon EMR Anda berada.

  4. Tambahkan aturan masuk.

    1. Untuk Tipe, pilih TCP Khusus.

    2. UntukSumber, pilihAdat.

    3. Masukkan nama grup keamanan Amazon Redshift Anda.

  5. Tambahkan aturan masuk lainnya.

    1. UntukJenis, pilihTCP.

    2. UntukRentang Pelabuhan, masuklah9083.

      catatan

      Port default untuk EMR HMS adalah 9083. Jika HMS Anda menggunakan port yang berbeda, tentukan port tersebut dalam aturan masuk dan dalam definisi skema eksternal.

    3. UntukSumber, pilihAdat.

  6. Masukkan nama dan deskripsi grup keamanan.

  7. Pilih Create security group (Buat grup keamanan).

Untuk menambahkan grup keamanan Amazon EC2 yang Anda buat dalam prosedur sebelumnya ke klaster Amazon Redshift Anda

  1. Di Amazon Redshift, pilih klaster Anda.

  2. Pilih Properti.

  3. LihatPengaturan jaringan dan keamanandan pilihlahMengedit.

  4. DalamGrup keamanan VPC, pilih nama grup keamanan baru.

  5. Pilih Save changes (Simpan perubahan).

Untuk menambahkan grup keamanan Amazon EC2 ke klaster Amazon EMR Anda

  1. Di Amazon EMR, pilih klaster Anda. Untuk informasi lebih lanjut, lihatMenggunakan konfigurasi keamanan untuk mengatur keamanan klasterdi dalamPanduan Manajemen Amazon EMR.

  2. Di bawahPerangkat Keras, pilih link untuk node Master.

  3. Pilih tautan diID instans EC2kolom.

  4. UntukTindakan, pilihKeamanan,Mengubah grup keamanan.

  5. DalamKelompok sercurity terkait, pilih grup keamanan baru, dan pilihTambahkan grup keamanan.

  6. Pilih Save (Simpan).
