Mengkueri data eksternal menggunakan Amazon Redshift Spectrum

Dengan menggunakan Amazon Redshift Spectrum, Anda dapat melakukan kueri dan mengambil data terstruktur dan semistruktur secara efisien dari file di Amazon S3 tanpa harus memuat data ke dalam tabel Amazon Redshift. Kueri Redshift Spectrum menggunakan paralelisme besar untuk berjalan sangat cepat terhadap kumpulan data besar. Sebagian besar pemrosesan terjadi di lapisan Redshift Spectrum, dan sebagian besar data tetap ada di Amazon S3. Beberapa klaster dapat secara bersamaan mengkueri kumpulan data yang sama di Amazon S3 tanpa perlu membuat salinan data untuk setiap klaster.

Ikhtisar Amazon Redshift Spectrum

Amazon Redshift Spectrum berada di server Amazon Redshift khusus yang tidak bergantung pada klaster Anda. Amazon Redshift mendorong banyak tugas intensif komputasi, seperti pemfilteran predikat dan agregasi, hingga ke lapisan Redshift Spectrum. Dengan demikian, kueri Redshift Spectrum menggunakan kapasitas pemrosesan klaster jauh lebih sedikit daripada kueri lainnya. Redshift Spectrum juga skala cerdas. Berdasarkan permintaan kueri Anda, Redshift Spectrum berpotensi menggunakan ribuan instance untuk memanfaatkan pemrosesan paralel secara besar-besaran.

Anda membuat tabel Redshift Spectrum dengan mendefinisikan struktur untuk file Anda dan mendaftarkannya sebagai tabel dalam katalog data eksternal. Katalog data eksternal dapatAWS Glue, katalog data yang disertakan dengan Amazon Athena, atau metastore Apache Hive Anda sendiri. Anda dapat membuat dan mengelola tabel eksternal baik dari Amazon Redshift menggunakan perintah bahasa definisi data (DDL) atau menggunakan alat lain yang terhubung ke katalog data eksternal. Perubahan pada katalog data eksternal segera tersedia untuk kluster Amazon Redshift Anda.

Opsional, Anda dapat partisi tabel eksternal pada satu atau lebih kolom. Mendefinisikan partisi sebagai bagian dari tabel eksternal dapat meningkatkan kinerja. Peningkatan terjadi karena pengoptimal kueri Amazon Redshift menghilangkan partisi yang tidak berisi data untuk kueri.

Setelah tabel Redshift Spectrum Anda ditentukan, Anda dapat melakukan kueri dan bergabung dengan tabel seperti halnya tabel Amazon Redshift lainnya. Redshift Spectrum tidak mendukung operasi pembaruan pada tabel eksternal. Anda dapat menambahkan tabel Redshift Spectrum ke beberapa klaster Amazon Redshift dan mengkueri data yang sama di Amazon S3 dari klaster mana pun dalam klaster yang samaAWSWilayah. Saat Anda memperbarui file data Amazon S3, data akan segera tersedia untuk kueri dari salah satu klaster Amazon Redshift Anda.

YangAWS GlueKatalog Data yang Anda akses mungkin dienkripsi untuk meningkatkan keamanan. JikaAWS Gluekatalog dienkripsi, Anda memerlukanAWS Key Management Service(AWS KMS) kunci untukAWS Glueuntuk mengaksesAWS Gluekatalog.AWS Glueenkripsi katalog tidak tersedia di semuaAWSDaerah. Untuk daftar yang didukungAWSDaerah, lihatEnkripsi dan Akses Aman untukAWS Gluedi dalamAWS GluePanduan Pengembang.Untuk informasi lebih lanjut tentangAWS GlueEnkripsi Katalog Data, lihatMengenkripsiAWS GlueKatalog Datadi dalamAWS GluePanduan Pengembang.

catatan

Anda tidak dapat melihat detail untuk tabel Redshift Spectrum menggunakan sumber daya yang sama dengan yang Anda gunakan untuk tabel Amazon Redshift standar, sepertiPG_TABLE_DEF,STV_TBL_PERM, PG_CLASS, atau information_schema. Jika intelijen bisnis atau alat analisis Anda tidak mengenali tabel eksternal Redshift Spectrum, konfigurasikan aplikasi Anda ke kueriSVV_EXTERNAL_TABLESdanSVV_EXTERNAL_COLUMNS.

Wilayah Amazon Redshift Spectrum

Redshift Spectrum tersedia diWilayah AWSdi mana Amazon Redshift tersedia, kecuali ditentukan lain dalam dokumentasi khusus Wilayah. UntukWilayah AWSketersediaan di Kawasan komersial, lihatEndpoint layananuntukAPI Pergeseran Merahdi dalamReferensi Umum Amazon Web.

Pertimbangan Amazon Redshift Spectrum

Perhatikan pertimbangan berikut saat Anda menggunakan Amazon Redshift Spectrum:

  • Klaster Amazon Redshift dan bucket Amazon S3 harus samaAWSWilayah.

  • Redshift Spectrum tidak mendukung perutean VPC yang disempurnakan dengan cluster yang disediakan. Untuk mengakses data Amazon S3, Anda mungkin perlu melakukan langkah-langkah konfigurasi tambahan. Untuk informasi lebih lanjut, lihatRedshift Spectrum dan perutean VPC yang disempurnakandi dalamPanduan Manajemen Amazon Redshift.

  • Redshift Spectrum mendukung alias titik akses Amazon S3. Untuk informasi lebih lanjut, lihatMenggunakan bucket—style alias untuk titik akses Andadi dalamPanduan Pengguna Layanan Penyimpanan Sederhana Amazon. Namun, Redshift Spectrum tidak mendukung VPC dengan alias titik akses Amazon S3. Untuk informasi lebih lanjut, lihatRedshift Spectrum dan perutean VPC yang disempurnakandi dalamPanduan Manajemen Amazon Redshift.

  • Anda tidak dapat melakukan pembaruan atau menghapus operasi pada tabel eksternal. Untuk membuat tabel eksternal baru dalam skema yang ditentukan, Anda dapat menggunakan CREATE EXTERNAL TABLE. Untuk informasi selengkapnya tentang CREATE EXTERNAL TABLE, lihatBUAT TABEL EKSTERNAL. Untuk memasukkan hasil query SELECT ke dalam tabel eksternal yang ada pada katalog eksternal, Anda dapat menggunakan INSERT (tabel eksternal). Untuk informasi selengkapnya tentang INSERT (tabel eksternal), lihatINSERT (tabel eksternal).

  • Kecuali Anda menggunakanAWS Glue Data Catalogyang diaktifkan untukAWSLake Formation, Anda tidak dapat mengontrol izin pengguna pada tabel eksternal. Sebagai gantinya, Anda dapat memberikan dan mencabut izin pada skema eksternal. Untuk informasi lebih lanjut tentang bekerja denganAWSFormasi Danau, lihatMenggunakan Redshift Spectrum denganAWS Lake Formation.

  • Untuk menjalankan kueri Redshift Spectrum, pengguna database harus memiliki izin untuk membuat tabel sementara dalam database. Contoh berikut memberikan izin sementara pada databasespectrumdbkepadaspectrumuserskelompok pengguna. 

    grant temp on database spectrumdb to group spectrumusers;

    Untuk informasi selengkapnya, lihat HIBAH.

  • Saat menggunakan Katalog Data Athena atauAWS GlueKatalog Data sebagai penyimpanan metadata, lihatKuota dan Batasdi dalamPanduan Manajemen Amazon Redshift.

  • Redshift Spectrum tidak mendukung Amazon EMR dengan Kerberos.
