Migration vers les vues de surveillance SYS

Lorsque vous migrez votre cluster provisionné Amazon Redshift vers Amazon Redshift sans serveur, vos requêtes de surveillance ou de diagnostic peuvent faire référence à des vues du système qui ne sont disponibles que sur les clusters provisionnés. Vous pouvez mettre à jour vos requêtes pour utiliser les vues de surveillance SYS. Cette page fournit des mappages de vues provisionnés vers SYS que vous pouvez consulter lors de la mise à jour de vos requêtes.

Cas d'utilisation pour la migration vers les vues de surveillance SYS

Migration à partir de clusters provisionnés vers Amazon Redshift sans serveur

Si vous migrez un cluster provisionné vers Amazon Redshift sans serveur, vous pouvez avoir des requêtes utilisant les vues système suivantes, qui ne sont disponibles que sur les clusters provisionnés.

  • Toutes les vues STL

  • Toutes les vues STV

  • Toutes les vues SVCS

  • Toutes les vues SVL

  • Certaines vues SVV

    • Pour obtenir la liste des vues SVV non prises en charge dans Amazon Redshift sans serveur, consultez la liste au bas de la section Surveillance des requêtes et des charges de travail avec Amazon Redshift sans serveur dans le Guide de gestion Amazon Redshift.

Pour continuer à utiliser vos requêtes, réajustez-les pour utiliser les colonnes définies dans les vues de surveillance SYS qui correspondent aux colonnes de vos vues provisionnées uniquement.

Mise à jour des requêtes tout en restant sur un cluster provisionné

Si vous ne migrez pas vers Amazon Redshift sans serveur, vous souhaiterez peut-être tout de même mettre à jour vos requêtes existantes. Conçues pour être faciles à utiliser et réduire la complexité, les vues de surveillance SYS fournissent une gamme complète de métriques pour une surveillance et un dépannage efficaces. En utilisant des vues SYS telles que SYS_QUERY_HISTORY et SYS_QUERY_DETAIL qui consolident les informations de plusieurs vues provisionnées uniquement, vous pouvez rationaliser vos requêtes.

Vous trouverez ci-dessous des listes de vues STL/SVCS/SVL/SVV avec une ou plusieurs colonnes également définies dans la vue de surveillance SYS donnée.
