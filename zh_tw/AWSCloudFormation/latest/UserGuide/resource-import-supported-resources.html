支援匯入和漂移偵測作業的資源

AWS CloudFormation 支援下列公共 (AWS) 資源類型的資源匯入和漂移偵測作業。

此外，CloudFormation 還支援可佈建私有資源類型的匯入作業；也就是說，佈建類型為 FULLY_MUTABLE 或 IMMUTABLE。若要匯入或執行私人資源類型的資源，您在帳號中註冊之資源類型的預設版本必須是可佈建的資源。如需有關資源佈建類型的詳細資訊，請參閱 AWS CloudFormation API 參考中的 DescribeType 動作與 AWS CLI 命令參考中的 DescribeType 命令的 ProvisioningType 參數。

如需有關漂移偵測的詳細資訊，請參閱偵測堆疊和資源未受管的組態變更。

注意

下列清單包含 FULLY_MUTABLE 或 IMMUTABLE 的佈建類型，而不是 NON_PROVISIONABLE 類型。FULLY_MUTABLE 和 IMMUTABLE 類型包含 (CRUDL) 處理常式，可以針對匯入和漂移偵測操作進行驗證，而 NON_PROVISIONABLE 類型不包含處理常式，並且無法針對匯入或漂移偵測操作進行驗證。
